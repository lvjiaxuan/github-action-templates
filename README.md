# Usage

## Release

Based on [lvr](https://github.com/lvjiaxuan/release).

Features:
1. Extracts release notes from `CHANGELOG.md` generated by lvr.
2. Supports syncing published packages to cnpm.

```yml
name: Release

on:
  push:
    tags:
      - v*

jobs:
  release:
    permissions:
      contents: write
      id-token: write
    uses: lvjiaxuan/github-action-templates/.github/workflows/release.yml@main
    with:
      publish: true
      sync_cnpm: true
```

### Inputs

| Input | Type | Default | Description |
| ----- | ---- | ------- | ----------- |
| `publish` | boolean | `false` | Whether to publish to NPM before releasing. |
| `sync_cnpm` | boolean | `false` | Whether to sync the published package to cnpm. |

## Update dependencies

> [!TIP]
> :point_right: [cron syntax help](https://crontab.guru/examples.html)

```yml
name: Update Dependencies

on:
  workflow_dispatch: {}
  schedule:
    - cron: 0 0 * * SAT

jobs:
  update-deps:
    uses: lvjiaxuan/github-action-templates/.github/workflows/update-deps.yml@main
```

### Inputs

| Input | Type | Default | Description |
| ----- | ---- | ------- | ----------- |
| `email` | string | `${{github.actor_id}}+${{github.actor}}@users.noreply.github.com` | The `user.email` config. |
| `commit_msg` | string | `chore: update deps on CI` | The commit message. |
| `mode` | string | `commit` | `commit` to push directly, or `pr` to create a pull request. |
